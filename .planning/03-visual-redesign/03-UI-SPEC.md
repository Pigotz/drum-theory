## Dark Theme Contract

This phase replaces the `seriph` theme (light background, `--slidev-theme-primary: #5d8392`) with a custom dark aesthetic.

**Implementation approach:** Override via a `style.css` at project root — Slidev auto-imports this file. Do NOT modify the theme package directly.

| Token | Value | Rationale |
|-------|-------|-----------|
| `--slidev-theme-primary` | `#e2e8f0` | Near-white for heading accent on dark surface |
| Slide background | `#0f1117` | Deep near-black — visually anchors SVG symbols and VexFlow stave |
| Surface background | `#1a1d27` | Slightly lighter dark for cards and side panels |
| Text primary | `#e2e8f0` | Off-white — readable, not harsh |
| Text secondary | `#94a3b8` | Muted slate — labels, captions |

**Critical constraint from STATE.md:** `seriph` theme was previously setting a global `ctx` white style that made VexFlow stave invisible. The fix is to ensure no global `color: white` or `fill: white` leak into VexFlow's SVG canvas context. VexFlow colors are controlled entirely via `setStyle()` / `setKeyStyle()` per-note in `DrumStaff.vue` — the CSS layer must not interfere.

## Spacing Scale

Slidev uses a 4-point base grid internally. This contract declares the spacing values used in custom component inline styles and the `style.css` override.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Compact element spacing, list item gaps |
| md | 16px | Default element padding (used in role cards, groove grid cells) |
| lg | 24px | Section padding (note duration panels, LoopPlayer container) |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major slide content breaks |
| 3xl | 64px | Page-level vertical centering margin |

Exceptions:
- Touch-target minimum: 44px min-height on LoopPlayer button (already present in `LoopPlayer.vue` — maintain on restyle)
- Groove grid cells: 24px min-width (existing value in `slides.md` table — maintain for legibility)
- VexFlow stave container: no padding override — DrumStaff.vue manages its own SVG dimensions (500×180px default)

## Typography

**Source:** CLAUDE.md references Dribbble design shots showing bold, heavy display typography. REQUIREMENTS TYPE-01/TYPE-02 mandate sharp, bold, on-brand headings and legible body text.

**Font stack decision:** Replace seriph's serif default with a bold grotesque. `Barlow Condensed` or `Space Grotesk` are on-brand for drumming/music contexts. Use Google Fonts import in `style.css`.

**Selected font:** Space Grotesk (weight 400 + 700) — geometric grotesque, bold at display size, compact at label size. Pairs with the industrial/angular feel of the Dribbble references.

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Display | 48px | 700 (bold) | 1.1 | Space Grotesk |
| Heading | 28px | 700 (bold) | 1.2 | Space Grotesk |
| Body | 18px | 400 (regular) | 1.5 | Space Grotesk |
| Label | 14px | 700 (bold) | 1.4 | Space Grotesk |

**Notes:**
- The existing `font-size: 48px; font-weight: 700` inline style on slide 8 (key insight slide) matches the Display role exactly — no change needed to that inline value.
- The existing `font-size: 28px; font-weight: 700` on note symbol headings (slides 3–7) matches the Heading role exactly — no change needed.
- The existing `font-size: 18px` on role cards (slide 11) and groove grid instructions matches the Body role exactly — no change needed.
- The existing `font-size: 14px` on groove grid table headers matches the Label role — no change needed.
- Only the font-family needs to change globally via `style.css`.

## SVG Note Symbol Contract

**Source:** REQUIREMENTS SVG-01/SVG-02, STATE.md, codebase inspection.

All five SVG files currently use `fill="white"` and `stroke="white"`. On the current `seriph` light theme this makes them invisible against the white background — they were designed for dark backgrounds and are compatible already.

**Action required:** No SVG file changes needed. The dark background will make them visible automatically. The STATE.md confirms: "SVG note symbols (slides 3–7) were designed with white stroke/fill — dark theme will make them compatible again without inversion."

**Proportion constraint:** SVG symbols are rendered at `height: 160px; width: auto` — maintain these dimensions. Do not scale or recolor the SVGs.

## Component Restyle Contracts

### LoopPlayer.vue

Current: indigo button (`#6366f1`) with emoji label.

Target on dark theme:

| State | Background | Text | Border |
|-------|-----------|------|--------|
| Play (idle) | `#e2e8f0` | `#0f1117` | none |
| Stop (active) | `#4b5563` | `#e2e8f0` | none |

- `border-radius: 6px` — maintain
- `padding: 12px 24px` — increase horizontal padding from 16px to 24px for presence
- `min-height: 44px` — maintain (touch target)
- `font-size: 14px; font-weight: 700` — maintain (Label role)
- Remove emoji from label text — use plain text: "Play" / "Stop"

### DrumStaff.vue

Current: VexFlow SVG renders on transparent background. Stave lines are black (VexFlow default), invisible on dark slide.

Target: Stave lines, clef symbol, and time signature must be `#e2e8f0`. This requires passing a context style to the VexFlow context before drawing.

Implementation pattern:
```js
ctx.setStrokeStyle('#e2e8f0')
ctx.setFillStyle('#e2e8f0')
// then draw stave
stave.setContext(ctx).draw()
// then restore per-note colors via setStyle() / setKeyStyle()
```

Note: per-note color calls happen AFTER the stave draw and BEFORE voice draw — existing pattern in DrumStaff.vue is already correct (step 6 in the component). The only addition is setting context stroke/fill before stave draw.

### Role Cards (slides.md, "Your Roles Tonight" slide)

Current: colored background cards with inline styles. These are correct and should be preserved exactly — the instrument colors (`#ef4444`, `#3b82f6`, `#eab308`) are legible on both light and dark backgrounds.

No changes needed to role card colors or typography.

### Groove Grid (slides.md, Human Drum slide)

Current: colored cells inline, table header text `color: white`. On dark background this text remains legible.

Target: Empty grid cells (the `padding: 8px` cells without a background color) should use `#1a1d27` (Secondary) instead of transparent to create visible cell boundaries on the dark slide background.

Add `border: 1px solid #2d3244` to all `<td>` elements for grid line visibility.
