  <action>
Run the deploy script from the project root:

```bash
cd /home/pigotz/Projects/drum-theory && npm run deploy
```

The `deploy` script from Plan 01 chains:
1. `npm run build` — runs `slidev build --base /drum-theory/`, outputs to `dist/`
2. `touch dist/.nojekyll` — prevents Jekyll from ignoring Vite's `_assets/` directory
3. `npx gh-pages -d dist` — pushes `dist/` to the `gh-pages` branch on GitHub

If `npm run deploy` fails:
- If build fails: check `slides.md` for syntax errors (`npm run build` alone to isolate)
- If gh-pages push fails with auth error: create a checkpoint for the user to authenticate
  with `gh auth login` and retry

After deploy completes:
1. Verify `dist/.nojekyll` exists: `ls dist/.nojekyll`
2. Verify `dist/index.html` exists: `ls dist/index.html`
3. Check the gh-pages branch was created:
   `git ls-remote origin gh-pages | grep gh-pages`

Then configure GitHub Pages via `gh` CLI (the Pages source must be set to gh-pages branch):

```bash
gh api repos/pigotz/drum-theory/pages \
  --method POST \
  --field source='{"branch":"gh-pages","path":"/"}' \
  2>/dev/null || echo "Pages may already be configured — check manually if this errors"
```

If the Pages API call returns an error (Pages already exists, or needs PUT instead of POST),
try:
```bash
gh api repos/pigotz/drum-theory/pages \
  --method PUT \
  --field source='{"branch":"gh-pages","path":"/"}' \
  2>/dev/null || echo "Check GitHub Settings → Pages manually"
```

Note to executor: The GitHub Pages API may require the gh-pages branch to exist first
(which the `npm run deploy` step creates), so run the API call AFTER deploy.
