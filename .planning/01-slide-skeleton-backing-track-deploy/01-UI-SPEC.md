# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Slide Skeleton + Backing Track + Deploy.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Context Note

This is a **Slidev presentation**, not a traditional web app. The "design system" is
Slidev's `seriph` theme (dark) plus inline Tailwind utility classes and scoped styles in
Vue components. There is no shadcn, no Radix, and no component registry.

All design decisions below apply to:
- Slide Markdown (`slides.md`) — layout, typography, content
- `LoopPlayer.vue` — the only interactive Vue component in Phase 1
- Groove grid (NOTE-04) — inline HTML color table on slide 12

## Interaction Contract

### v-click Reveals (SLIDE-02, SLIDE-03)

- Slides 3–7: left column shows note symbol on load; right column (pizza analogy) appears on first click
- Slide 11: volunteer role cards appear one per click — kick on click 1, snare on click 2, hi-hat on click 3
- No custom animation needed — Slidev default v-click fade-in is correct
- Do NOT use `v-motion` or third-party animation in Phase 1

### LoopPlayer.vue (AUDIO-01, AUDIO-02)

- Button shows "▶ Play" on initial load
- On click: `await Tone.start()` first, then toggle audio — button shows "⏹ Stop"
- Audio loops via HTML5 `<audio loop>` attribute (no Tone.js needed for Phase 1 backing track)
- No autoplay — browser gesture required, enforced by design
- Button state is local to the component — no global state
- Audio source constructed as: `import.meta.env.BASE_URL + 'backing.mp3'`

### Groove Grid (NOTE-04)

- Static HTML `<table>` — no interactivity
- Beat header row: labels "1 + 2 + 3 + 4 +" in white text on dark background
- Hi-hat row: cells with hit = `background: #eab308`, rest = transparent/dark
- Snare row: cells with hit = `background: #3b82f6`, rest = transparent/dark
- Kick row: cells with hit = `background: #ef4444`, rest = transparent/dark
- Row labels ("HH", "SN", "KK") in white, left-aligned, weight 600

## Presenter View Contract (SLIDE-04, SLIDE-05)

- Speaker notes present on every slide via `<!-- note -->` Slidev comment blocks
- Notes include: spoken prompts, timing cues, audience interaction reminders
- Countdown timer: 12 minutes, configured via Slidev `timer` frontmatter option

## Accessibility Notes

- LoopPlayer button must have descriptive `aria-label` updated on state change:
  - Playing: `aria-label="Stop backing track"`
  - Stopped: `aria-label="Play backing track"`
- Color alone does not convey groove grid meaning — row labels ("HH", "SN", "KK") provide text alternative
- Presentation context: projector display, not screen reader target — WCAG AA contrast is a best-effort goal, not a hard blocker
