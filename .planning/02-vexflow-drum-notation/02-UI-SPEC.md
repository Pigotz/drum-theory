> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Context Note

This is a **Slidev presentation**, not a traditional web app. Phase 2 adds one new Vue
component — `DrumStaff.vue` — that renders a VexFlow SVG percussion stave inside slide 9.
The component replaces the `image: /drum-sheet.png` placeholder on the `image-right` slide.

All design decisions below apply to:
- `DrumStaff.vue` — the new VexFlow SVG component
- Slide 9 (`image-right` layout) — where `DrumStaff.vue` replaces the static image panel
- Any other notation diagram slides if added

All Phase 1 design decisions (spacing scale, typography, color, groove grid, LoopPlayer.vue)
are **inherited unchanged**. This spec covers only the delta introduced by Phase 2.

**Source of inherited decisions:** Phase 1 UI-SPEC
(`01-slide-skeleton-backing-track-deploy/01-UI-SPEC.md`)

### VexFlow SVG Coloring Contract

VexFlow 5.x allows per-note SVG style override. The coloring contract for `DrumStaff.vue`:

| Element | Color | Method |
|---------|-------|--------|
| Hi-hat notehead (x-notehead) | `#eab308` | `note.setStyle({ fillStyle: '#eab308', strokeStyle: '#eab308' })` |
| Hi-hat stem | `#eab308` | Included in `setStyle` — stroke applies to stem SVG path |
| Snare notehead (standard oval) | `#3b82f6` | `note.setStyle({ fillStyle: '#3b82f6', strokeStyle: '#3b82f6' })` |
| Snare stem | `#3b82f6` | Included in `setStyle` |
| Kick notehead (standard oval) | `#ef4444` | `note.setStyle({ fillStyle: '#ef4444', strokeStyle: '#ef4444' })` |
| Kick stem | `#ef4444` | Included in `setStyle` |
| Stave lines | `white` or theme text color | Inherit VexFlow default; do not override unless illegible on dark bg |
| Percussion clef | `white` or theme text color | Inherit VexFlow default |
| Barline | `white` or theme text color | Inherit VexFlow default |

**Critical:** `setStyle` must be called BEFORE `vf.draw()`. Calling it after draw has no
effect in VexFlow 5.x SVG backend.

## DrumStaff.vue Component Contract

### Props

| Prop | Type | Required | Default | Description |
|------|------|----------|---------|-------------|
| `pattern` | `DrumPattern` | yes | — | Beat pattern to render (see interface below) |
| `width` | `number` | no | `500` | SVG canvas width in px; passed to VexFlow renderer |
| `height` | `number` | no | `180` | SVG canvas height in px; passed to VexFlow renderer |

### DrumPattern Interface

```ts
interface DrumBeat {
  instrument: 'kick' | 'snare' | 'hihat'
  // Staff position string for VexFlow percussion stave.
  // hi-hat: 'a/5' (above staff, x-notehead)
  // snare:  'c/5' (third space)
  // kick:   'e/4' (below staff)
  position: string
  // VexFlow note duration string: 'q' = quarter, '8' = eighth, 'h' = half, 'w' = whole
  duration: string
  // x-notehead for hi-hat; omit for kick/snare (standard oval)
  noteType?: 'x'
}

interface DrumPattern {
  // Time signature numerator (default: 4)
  beats?: number
  // Beat units (default: 4 = quarter note)
  beatValue?: number
  // Voices: voice 1 = hi-hat/snare (stems up), voice 2 = kick (stems down)
  voiceUp: DrumBeat[]   // hi-hat and snare
  voiceDown: DrumBeat[] // kick only
}
```

### Staff Position Defaults

These are the canonical positions for the basic rock groove (Groove 1):

| Instrument | Staff Position | Note Type | Voice | Stem |
|------------|---------------|-----------|-------|------|
| Hi-hat | `a/5` | `x` (x-notehead) | voiceUp | up |
| Snare | `c/5` | standard | voiceUp | up |
| Kick | `e/4` | standard | voiceDown | down |

Note: These positions must be verified empirically during the VexFlow spike (see ROADMAP.md
open questions). The values above are best-estimate from standard percussion clef conventions
and must be confirmed against VexFlow 5.0.0 output before locking the component API.

### Initialization Pattern

```vue
<script setup>
import { ref, onMounted } from 'vue'

const container = ref(null)

onMounted(() => {
  if (!container.value) return   // guard null ref — mandatory per CLAUDE.md

  // VexFlow initialization goes here
  // const vf = new Vex.Flow.Factory({ renderer: { elementId: container.value, ... } })
})
</script>

<template>
  <div ref="container" />
</template>
```

**Lifecycle rule:** All VexFlow initialization inside `onMounted()`. Guard null refs.
Source: CLAUDE.md (locked).

**Slidev lifecycle note:** If `DrumStaff.vue` needs to re-render on slide re-entry (e.g.
presenter navigates back to slide 9), use `onSlideEnter` from `@slidev/client` in addition
to `onMounted`. Do NOT rely on `onMounted` alone for re-entry behavior. Source: CLAUDE.md.

### Container Sizing

- `DrumStaff.vue` renders into a `<div ref="container">` — VexFlow uses this as the
  SVG mount point
- On the `image-right` layout, the right panel is approximately 50% of slide width
  (~480–520px at 1280px slide width)
- Default `width` prop: `500px` — safe fit for the image-right panel on a 1280px slide
- Default `height` prop: `180px` — single stave with percussion clef; provides visual
  breathing room without crowding the slide
- Do NOT add `overflow: hidden` to the container — VexFlow may render stave elements
  slightly outside the declared dimensions

## Slide 9 Layout Contract

Slide 9 is updated in Phase 2 from a static image to a live VexFlow component.

| Property | Phase 1 Value | Phase 2 Value | Delta |
|----------|--------------|--------------|-------|
| Layout | `image-right` | `default` (or custom panel) | Changed — image-right cannot mount a Vue component in the image slot |
| Right panel content | `image: /drum-sheet.png` | `<DrumStaff :pattern="groove1" />` | Replaced |
| Left panel content | Color-coded bullet list | Same — no change | Unchanged |
| Speaker notes | Placeholder note present | Update note: remove placeholder reference | Updated |

**Layout migration note:** Slidev's `image-right` layout uses a static `image:` frontmatter
key — it cannot mount a Vue component in the image panel. Phase 2 must either:
1. Use `layout: two-cols` with the right column containing `<DrumStaff />`, or
2. Use `layout: default` with manual two-column CSS flexbox inside the slide

Recommendation: Use `layout: two-cols` — it matches the existing pattern from Phase 1
(slides 3–7) and the left column content (bullets) is already in the left panel position.
The right column receives `<DrumStaff :pattern="groove1" />`.

### Groove 1 Pattern (Basic Rock)

The DrumPattern object to pass as the `:pattern` prop on slide 9:

```ts
const groove1: DrumPattern = {
  beats: 4,
  beatValue: 4,
  voiceUp: [
    // Hi-hat: every eighth note (8 × eighth notes in 4/4)
    { instrument: 'hihat', position: 'a/5', duration: '8', noteType: 'x' },
    { instrument: 'hihat', position: 'a/5', duration: '8', noteType: 'x' },
    { instrument: 'hihat', position: 'a/5', duration: '8', noteType: 'x' },
    { instrument: 'hihat', position: 'a/5', duration: '8', noteType: 'x' },
    { instrument: 'hihat', position: 'a/5', duration: '8', noteType: 'x' },
    { instrument: 'hihat', position: 'a/5', duration: '8', noteType: 'x' },
    { instrument: 'hihat', position: 'a/5', duration: '8', noteType: 'x' },
    { instrument: 'hihat', position: 'a/5', duration: '8', noteType: 'x' },
    // Snare on beats 2 and 4 (voiced together with hi-hat as chords or separate notes)
    // Implementation detail: use VexFlow voice merging or chord notation at beat 2 and 4
  ],
  voiceDown: [
    // Kick on beats 1 and 3 (quarter notes at position 1 and 3 of 4/4)
    { instrument: 'kick', position: 'e/4', duration: 'q' },
    { instrument: 'kick', position: 'e/4', duration: 'q', rest: true },  // beat 2 rest
    { instrument: 'kick', position: 'e/4', duration: 'q' },
    { instrument: 'kick', position: 'e/4', duration: 'q', rest: true },  // beat 4 rest
  ]
}
```

Note: The exact VexFlow voice merging API for combining snare with hi-hat on the same
beat must be resolved in the spike. The pattern above is a design-intent specification,
not final VexFlow API calls.

## Interaction Contract

### DrumStaff.vue (NOTE-01, NOTE-02, NOTE-03)

- Component is **static** — no interactivity in Phase 2 (PlayableDrumStaff.vue with Tone.js
  audio cursor is v2 scope)
- No click handlers, no play button, no hover states
- Renders once on `onMounted`; re-renders on `onSlideEnter` if needed
- SVG is display-only — do NOT add `tabindex`, role, or keyboard handlers (no interaction)

### Inherited interactions (unchanged from Phase 1)

- Slide 9 has no v-click reveals — the drum notation is shown in full on slide entry
- LoopPlayer.vue on slide 12 is unchanged
- Groove grid on slide 12 is unchanged

## Accessibility Notes

- `DrumStaff.vue` container `<div>` should carry `role="img"` and
  `aria-label="Drum notation: basic rock groove with kick on beats 1 and 3, snare on 2 and 4, hi-hat every eighth note"`
- VexFlow SVG does not generate accessible text by default — the `aria-label` on the
  container compensates
- Color alone does not convey instrument identity — the left-panel bullet list on slide 9
  provides the text mapping (hi-hat/snare/kick with names)
- Presentation context: projector display, not screen reader target — WCAG AA contrast
  is best-effort, not a hard blocker
